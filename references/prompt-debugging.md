# Prompt 诊断框架

当用户的生图结果不理想时，用这个诊断树系统性地找出问题根因——而不是盲目重写。

---

## 诊断树

从用户描述的"出图问题"出发，按层级排查：

```
用户反馈："生成的图片不对"
        │
        ├─→ 问题类型 1：【主体错误】
        │   ├─ 主体完全没出现 → Token 权重不够 / 被其他 token 压制
        │   │   → 解决：将该主体移到 prompt 最前面 / 减少无关内容 / 提高权重
        │   ├─ 主体出现但特征错 → 属性绑定问题
        │   │   → 解决：检查组句是否让形容词和名词清晰配对 / 用空间描述隔离
        │   └─ 多出一个不该有的主体 → 训练数据关联/概念泄漏
        │       → 解决：加负向 prompt 排除 / 换更干净的描述词
        │
        ├─→ 问题类型 2：【风格跑偏】
        │   ├─ 完全不是想要的风格 → 风格 token 被 content token 压制
        │   │   → 解决：加强风格 token 权重 / 减少风格方向的数量（只保留一个）
        │   ├─ 风格混杂/四不像 → 多个冲突的风格 token 取平均
        │   │   → 解决：只保留一个核心风格 / 把风格描述合并为更具体的单一标签
        │   └─ 风格正确但"味道不够" → 风格 token 太泛
        │       → 解决：用更具体的风格词替代笼统词（"Studio Ghibli 1980s cel animation" > "anime"）
        │
        ├─→ 问题类型 3：【构图/布局混乱】
        │   ├─ 主体位置不对 → 纯 prompt 难以精确控制位置
        │   │   → 解决：加空间描述词 / 明确告知用户纯 prompt 的局限性
        │   ├─ 比例失调 → 缺少参照物或尺度描述
        │   │   → 解决：加入参照物或明确尺寸描述
        │   └─ 画面太满/太空 → 缺少负空间或背景密度描述
        │       → 解决：加 "negative space" / "minimalist composition" 或相反
        │
        ├─→ 问题类型 4：【画质问题】
        │   ├─ 模糊/不清晰 → CFG 过低 / 步数不够 / 负向 prompt 弱
        │   │   → 解决：建议提高 CFG / 增加步数（不用改 prompt）
        │   ├─ 过度锐化/不自然 → CFG 过高
        │   │   → 解决：建议降低 CFG（不用改 prompt）
        │   └─ 细节缺失 → prompt 太泛、缺少具体纹理描述
        │       → 解决：加入材质、纹理等具象描述
        │
        └─→ 问题类型 5：【"感觉不对"——最模糊的一类】
            ├─ 氛围/情绪不对 → 光线和色彩 token 不够明确
            │   → 解决：加入具体的光线和色调描述
            ├─ "太假/太AI感" → 缺少生活感/不完美描述
            │   → 解决：加入 "film grain", "snapshot aesthetic", "natural imperfections"
            └─ "就是没有那个味道" → 缺少文化/美学锚点
                → 解决：加入具体的艺术家、电影、摄影师作为风格锚点
```

## 快速诊断表

不需要用户描述——直接从 prompt 文本本身就能预判潜在问题：

| Prompt 特征 | 预示的风险 | 预防措施 |
|------------|-----------|---------|
| **主体词在第 10 个 token 之后** | 主体可能不够突出 | 把最重要的主体移到最前面 |
| **超过 3 个独立主体** | 属性绑定混乱概率大增 | 减少主体或加入空间分隔词 |
| **3 个以上不同风格标签** | 风格互相稀释/抵消 | 合并为一个明确的风格方向 |
| **大量抽象形容词（beautiful, amazing, wonderful）** | 这些词 embedding 接近噪声 | 替换为具象的视觉描述 |
| **没有光线/色彩描述** | 画面可能平淡 | 至少加入一个光线或色调 token |
| **质量词（8K, masterpiece）占比过高** | 浪费 token 预算 | 减少或删除，用具体纹理替代 |
| **负向 prompt 超过 50 个 token** | 稀释有效词的推力 | 精简到 ≤ 15 个核心负向 token |
| **中英文混用** | 编码质量下降 | 统一用英文（除非用 T5 系模型且中文支持好） |

## 平台特有的诊断补充

### SD 1.5 专属
- prompt 超过 77 token → **检查是否被截断**！被截断的部分完全不起作用
- 权重括号嵌套过多 → 检查 `(word:1.5)` 是否合理，过高的权重产生 artifacts

### Midjourney 专属
- `--stylize` 过高 → 画面过于"MJ感"，失去用户想要的风格
- `--chaos` 过低 → 四张图太相似，多样性不足
- 用了 `--style raw` 但 prompt 太短 → raw 模式需要更详细的描述

### Flux / SD3 专属
- 不需要逗号分隔的标签风格——写了反而可能不如自然语言
- 对 prompt 的跟随力极强，如果效果不对很可能是 prompt 本身的描述有问题

### DALL-E 3 / ChatGPT 生图 专属
- prompt 会被内部改写——太复杂的描述可能被简化
- 利用多轮对话迭代：不要一次给全部要求，逐步添加

## 诊断方法论

当帮助用户诊断一个问题 prompt 时，按以下程序：

### Step 1: 解析 Prompt 的 Token 组成

快速将 prompt 的每个成分归类：

```
Original: "a cute cat sitting on a beautiful chair, 8K, highly detailed, masterpiece"

Content: cat (位置2), chair (位置8) → 弱！chair 太靠后
Style: cute (主观,弱), beautiful (噪声) → 弱！
Structure: sitting on (基本可用) → 中等
Quality: 8K, highly detailed, masterpiece → 无用
```

### Step 2: 识别最弱的一环

是 content 不够具体？style 太模糊？还是 structure 完全缺失？

### Step 3: 给出针对性修复

不是重写整个 prompt——精准增强最弱的部分。

### Step 4: 如果仍然不行

考虑是否是**模型/参数层面**的问题（CFG、步数、分辨率），而不是 prompt 的问题。在建议用户改 prompt 之前，先建议检查这些参数。
